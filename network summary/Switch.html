Switching

(MAC 테이블에 주소가 없다면)
Learning: 모르는 MAC 주소(수신지)로부터 프레임이 수신되면 모든 포트에 해당 MAC 주소를 전송(flooding)하고, 해당 port와 MAC 주소의 연관성을 담은 MAC 테이블을 갱신합니다.

(MAC 테이블에 주소가 있다면)
Filtering: 수신된 프레임의 목적지가 수신된 포트안에 포함된다면, 이 프레임을 무시합니다.
Forwarding: 수신된 프레임의 목적지가 다른 포트로 이어진다면, 두 포트를 연결합니다.

VLAN(Virtual LANs)

3계층에서 라우터를 통해 물리적인 LAN을 구축한다면, 2계층의 스위치에서는 논리적인 LAN(subnet)을 구축합니다.
같은 네트워크 주소를 사용하고 있어도 VLAN으로 분리하면 (서로 다른 네트워크 주소를 사용하는 것처럼) 브로드캐스트 도메인이 분리됩니다. (라우터에 LAN카드 하나를 추가하는데는 큰 비용이 드는데 반해 스위치의 VLAN을 추가하는데 따로 비용이 들지 않아 많이 사용됩니다.)
1.VLAN 생성
- #vlan database
- (vlan)#vlan[number]
 2.Port 연결
- (config)#interface [Interface Type Slot/Port]
- (config-if)#switchport access vlan[number]
 3.Trunk 설정
(VLAN마다 Link를 따로 두지 않고 Trunk로 지정된 하나의 물리적 Link 안에 여럿의 논리적 Link를 구축합니다.)
- (config)#interface [Inteface Type Slot/Port]
- (config-if)#switchport mode trunk

스위치의 IP

:일반적으로 2계칭 장비인 스위치의 인터페이스에는 ip 할당이 불가능합니다. 때문에 스위치 자체에 ip를 할당할 때는 기본 가상 인터페이스인 vlan1에 ip를 할당하여 사용합니다.

데이터 전송 방식

-Simplex: 일방향 전송(TV)
-Half-Duplex: 하나의 경로로 양방향 통신, 동시 전송 불가(무전기)
-Full-Duplex: 각자의 경로로 양방향 통신, 동시 전송 가능(핸드폰)

Flooding

모든 포트로 같은 프레임을 복사하여 보내는 것을 말합니다. 복사하여 보내기 때문에 프레임을 넘치게 보낸다는 의미에서 flooding이라 하는 것 같습니다.

delay & latency

delay: 지정된 시간보다 지체된 시간
latency: 한 지점에 머무는 시간
