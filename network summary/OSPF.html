OSPF(Open Shortest Path First Protocol)

특징

Link state Routing 알고리즘을 사용합니다.

짧은 수렴시간(Short Convergence Time): 수렴시간이 짧아 무한루프가 발생하지 않습니다.

VLSM(Variable Length Subnet Mask): 다양한 영역의 서브넷 마스크를 제공합니다.(classless)

네트워크 크기 제한 없음: 수렴시간이 짧아 무한루프에 대비한 최대 hop 수가 없어 네트워크 크기에 제한도 없습니다.

낮은 네트워크 대역폭 활용: Hello 메세지를 통한 생사확인 이외에는 변화되는 정보만 교환해 정보 교환의 크기와 횟수가 적어 대역폭도 감소합니다.

라우팅 명령어

- (config)#router ospf [Process ID]
- (config-router)#network [(Directly Connected)Network-Address] [Wildcard-Mask] area[Area-Number]

Router ID

Router는 인터페이스별로 IP를 가지고 있기 때문에 Router 자신을 가리키는 위해서는 Router ID가 필요합니다.

우선 선출 순서

  1. 관리자가 직접 설정한 경우
    - (config)#router ospf [Process-ID]
    - (config-router)#router-id [~.~.~.~]
  2. Loopback I/F가 있는 경우
    : Loopback Address 중 숫자가 가장 큰 IP 사용
  3. 기본
    : 자신의 인터페이스 IP 중 숫자가 가장 큰 IP 사용(+Unique)

(동작 과정)

생각보다 동작 과정이 복잡하여 현재 수준에서 정리하기 어려움, 아래 정리 내용은 인터넷상의 정보 요약.

1. Down State
-OSPF Hello를 (멀티캐스트:224.0.0.5로) 전송하고 있는 상태

2. Init State
-Hello를 수신, 수신한 Hello의 Neighbor List에 수신자의 Router-ID가 없는 상태

3. Attempt State(NBMA)

4. Two-Way State
-서로의 Router-ID 교환, DR/BDR 선출
-LAN (4-1. Point to Point WAN: 각자를 DR로 지정
4-2. LAN: 내부에서 RouterID + 우선순위가 큰 순서로 DR/BDR 선출)

5. Exstart State
-Master/Slave 성립 (Router ID가 가장 큰 라우터가 Master)

6. Exchange State
-DBD 교환

7. Loading State
-(LSR LSU 송수신 시)LSDB 갱신

8.Full State
-SPF(Short Path First)알고리즘으로 최적 경로 계산
-Area내, 모든 라우터의 LSDB 동기화

Packet Type

1. Hello Packet
: 이웃 관계 성사, 이웃 관계 유지(생사 확인)

2. DBD(DataBase Description Packet)
: 인접 이웃에게 요약된 자신의 LSA 전송

3. LSR(Link State Request Packet)
: 받은 DBD에서 모르는 N/W 정보 발견시, 상대 라우터에게 상세한 LSA 요구

4. LSU(Link State Update Packet)
: 요청된 LSR에 대한 응답 혹은 자신의 링크 상태 변경시 광고

5. LSAck(Link State Acknowledgement)
: DBD, LSR, LSU에 대한 정상 수신 확인

Link state Routing Protocol

:네트워크 전체 토폴로지에 대한 DB를 모든 라우터가 갖고 있으며, 변경된 네트워크 상황에 대해서만 이웃 라우터에게 알립니다.

LSA(Link State Advertisement)

:OSPF에서 사용하는 '링크 상태 정보', 여러 종류의 타입으로 나뉜다.
