1. IP 주소 할당

연결을 원하는 각각의 인터페이스에 논리 주소를 할당합니다.
- (config)#interface [Interface Type Slot/Port]
- (config-if)#ip address [Ip-Address] [Subnet-mask]

1-1. Inter-VLAN

라우터의 입장에서 VLAN은 하나의 물리적 link(Trunk)로 여럿의 논리적 link가 들어오는 꼴입니다. 즉 하나의 interface로 들어온 여럿의 N/W에게 어떻게 IP를 부여해야할지 곤란해집니다. 여기서 나온 개념이 subinterface입니다. 하나의 물리적 링크 안에 여럿의 논리적 링크가 있는 것에 대응하여 하나의 물리적 인터페이스 안에 여럿의 논리적 인터페이스를 두는 것입니다.
1.subinterface를 구축합니다.
- (config)#interface [Interface Type Slot/Port].[Number(0~255)]
 2.연결할 VLAN의 tag를 붙입니다.
(트렁크를 지나는 프레임이 자신의 VLAN을 찾아갈 수 있도록 추가 헤더를 만들어주는 설정입니다.)
 - (config-subif)#encapsulation dot1q [VLAN Number]
 3.IP 주소를 부여합니다.
- (config-subif)#ip address [Ip Address] [Subnet Mask]

2. Routing

자신의 인터페이스에 없는 다른 네트워크(브로드캐스트 범위 밖)의 정보를 찾아 기록하고, 다양한 이동 경로 중에서 최적의 이동 경로를 선정합니다.

Static Routing
:관리자가 직접 모르는 주소를 찾아서 기록하고, 모든 이동 경로를 선정합니다.
- (config)#ip route [Destination-Network-Address] [subnet-mask] [Next-hop-Address | Exit I/F] [Administrative Distance]


Dynamic Routing
:프로토콜(알고리즘)이 모르는 주소를 탐색하고 최적의 이동 경로를 선정합니다.
(사용하는 프로토콜에 따라 명령어의 내용이 달라집니다. 자세한 사항은 왼쪽 상단의 목록에서 확인해주세요.)

3. ACL(Access Control Lists)

라우터에서 설정 가능한 간단한 보안 기능입니다. 트래픽에 대한 접근 제어를 정의하고 그 내용을 인터페이스에 저장합니다.

Standard ACL: 라우터의 인터페이스에서 특정 IP에 대한 접근을 제한시킬 수 있습니다.
- (config)#access-list [Access-List-Number(1~99)] {Permit/Deny} [Source-Address] [Source-Wildcard-Mask]

Extended ACL: 라우터의 인터페이스에서 특정 IP 및 특정 프로토콜에 대한 접근을 제한시킬 수 있습니다.
- (config)#access-list [Access-List-Number(100~199)] {permit/Deny} [Protocol] [Source-Address] [Source-Wildcard-Mask] [Destination-Address] [Destination-Wildcard-Mask] [Operator(gt,lt,eq,neq)] [Port]

인터페이스에 ACL 연결
- (config)#interface [Interface Type Slot/Port]
- (config-if)# [protocol] access-group [Access-list-Number(1~199)] {in/out}

3-1. Wildcard-Mask

Subnet-Mask가 서브넷팅된 네트워크 주소 영역을 추출하기 위함이라면 Wildcard-Mask는 특정 IP 주소를 추출하기 위한 것입니다. (Subnet-Mask는 연속적인 0,1 표현만이 가능하기 때문에 특정 주소에 대한 표현이 어려워 불연속적 표현이 가능한 Wildcard-Mask를 따로 사용합니다.)
0의 의미: 값이 고정되는 부분(일치해야하는 부분)
1의 의미: 값이 바뀌어도 되는 부분(일치할 필요가 없는 부분)

예시:)
IP: 146.34.48.0/23
Wildcard-Mask: 0.0.00000001.255

NAT(Network Address Translation)

NAT는 내부에서 외부로 나가는 IP 패킷의 TCP/UDP 포트 및 목적지 IP 등을 재기록합니다. 이 과정에서 다수의 사설 주소를 하나의 공인 주소로 번역하여 송출합니다.

Static 설정(1:1 매칭)
1.NAT를 설정합니다.
- (config)#ip nat inside source static [Inside IP-Address] [Outside IP-Address]
 2.각 인터페이스에서 내부와 외부를 각각 지정합니다.
- (config)#interface [Interface Type Slot/Port]
- (config-if)#ip nat [inside/outside]

Dynamic 설정(N:N 매칭)
1.NAT를 사용할 내부 주소 범위를 지정합니다.
- (config)#access-list [Number] permit [Inside IP-Address] [Wildcard-Mask]
 2.NAT를 사용할 외부 주소 범위를 지정합니다.
- (config)#ip nat pool [Name] [Start IP-Address] [End IP-Address] netmask [Subnet-Mask]
 3.NAT를 설정합니다.
- (config)#ip nat inside source list [Number] pool [name]
 4.각 인터페이스에서 내부와 외부를 각각 지정합니다.
- (config)#interface [Interface Type Slot/Port]
- (config-if)#ip nat [inside/outside]

PAT(port address translation) 설정(N:1 매칭)
1.NAT를 사용할 내부 주소 범위를 지정합니다.
- (config)#access-list [Number] permit [Ip-address] [Wildcard-Mask]
 2.NAT를 설정합니다.
- (config)#ip nat inside source list [Number] interface [Outside Interface Type Slot/Port] overload
 3.각 인터페이스에서 내부와 외부를 각각 지정합니다.
- (config)#interface [Interface Type Slot/Port]
- (config-if)#ip nat [inside/outside]

*interface

:매체 사이의 출입구를 가리킵니다. 하나의 노드가 외부와 처음으로 접촉하는 지점입니다. (서로 다른 매체의 인터페이스 두 개가 연결되면 하나의 통로가 완성됩니다.)

*R/T(Routing Table)

:해당 라우터가 현재 알고 있는 네트워크(주소) 목록을 가리킵니다.

Gateway

:LAN과 WAN을 연결해주는(Ethernet을 Internet으로 연결) 관문의 역할을 하는 인터페이스를 가리킵니다.

dot1q

VLAN의 encapsulation(tagging) 방식 중 하나입니다. IEEE802.1q와 ISL 두가지 방식이 있는데 ISL은 이제는 잘쓰이지 않고 IEEE802.1q를 명령어상에서는 dot1q로 표기합니다.

*ACL 용어 정리

any: 모든 네트워크를 대상으로 할 때
host: 단일 주소를 대상으로 할때

공인&사설 주소

공인 주소: 유료, 유일한 IP, 인터넷 가능
사설 주소: 무료, 겹치는 IP, 인터넷 불가
