Router

1. IP 주소 할당

연결을 원하는 각각의 인터페이스에 논리 주소를 할당합니다.
- (config)#interface [Interface Type Slot/Port]
- (config-if)#ip address [Ip-Address] [Subnet-mask]

2. Routing

다른 네트워크(브로드캐스트 범위 밖)의 정보를 찾아 기록하고, 다양한 이동 경로 중에서 최적의 이동 경로를 선정합니다.

Static Routing
:관리자가 직접 모르는 주소를 찾아서 기록하고, 모든 이동 경로를 선정합니다.
- (config)#ip route [Destination-Network-Address] [subnet-mask] [Next-hop-Address | Exit I/F] [Administrative Distance]


Dynamic Routing
:프로토콜(알고리즘)이 모르는 주소를 탐색하고 최적의 이동 경로를 선정합니다.
(사용하는 프로토콜에 따라 명령어의 내용이 달라집니다. 자세한 사항은 왼쪽 상단의 목록에서 확인해주세요.)

3. ACL(Access Control Lists)

라우터에서 설정 가능한 간단한 보안 기능입니다. 트래픽에 대한 접근 제어를 정의하고 그 내용을 인터페이스에 저장합니다.

Standard ACL: 라우터의 인터페이스에서 특정 IP에 대한 접근을 제한시킬 수 있습니다.
- (config)#access-list [Access-List-Number(1~99)] {Permit/Deny} [Source-Address] [Source-Wildcard-Mask]

Extended ACL: 라우터의 인터페이스에서 특정 IP 및 특정 프로토콜에 대한 접근을 제한시킬 수 있습니다.
- (config)#access-list [Access-List-Number(100~199)] {permit/Deny} [Protocol] [Source-Address] [Source-Wildcard-Mask] [Destination-Address] [Destination-Wildcard-Mask] [Operator(gt,lt,eq,neq)] [Port]

인터페이스에 ACL 묶기
- (config)#interface [Interface Type Slot/Port]
- (config-if)# [protocol] access-group [Access-list-Number(1~199)] {in/out}

3-1. Wildcard-Mask

Subnet-Mask가 서브넷팅된 네트워크 주소 영역을 추출하기 위함이라면 Wildcard-Mask는 특정 IP 주소를 추출하기 위한 것입니다. (Subnet-Mask는 연속적인 0,1 표현만이 가능하기 때문에 특정 주소에 대한 표현이 어려워 불연속적 표현이 가능한 Wildcard-Mask를 따로 사용합니다.)
0의 의미: 값이 고정되는 부분(일치해야하는 부분)
1의 의미: 값이 바뀌어도 되는 부분(일치할 필요가 없는 부분)

예시:)
IP: 146.34.48.0/23
Wildcard-Mask: 0.0.00000001.255

*interface

:매체 사이의 출입구를 가리킵니다. 한 매체가 외부와 처음으로 접촉하는 지점입니다. (서로 다른 매체의 인터페이스 두 개가 연결되면 하나의 통로가 완성됩니다.)

*R/T(Routing Table)

:해당 라우터가 현재 알고 있는 네트워크(주소) 목록을 가리킵니다.

Gateway

:LAN과 WAN을 연결해주는(Ethernet을 Internet으로 연결) 관문의 역할을 하는 인터페이스를 가리킵니다.
