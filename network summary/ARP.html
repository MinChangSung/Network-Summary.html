ARP(Address Resolution Protocol)

소개

ARP: (상대방의) MAC 주소를 모를 때 (상대방의) IP 주소를 통해 (상대방의) MAC 주소를 알려주는 서비스입니다.
(MAC주소를 모르면 프레임을 형성할 수 없으므로 데이터 전송을 위해서는 MAC주소가 필수적입니다.)

IP 주소를 이더넷 주소로 변환

설명

LAN 전체에 Broadcast로 상대방의 IP주소를 요청(ARP Request 패킷 전송)하면 해당 IP의 상대방이 자신의 MAC주소와 함께 응답(ARP Reply 패킷 전송)을 보냅니다.

IP 기반 프로토콜

  • ICMP
  • ARP
  • IGMP

*RARP (Reverse ARP)

:(자신의) IP 주소를 모를 때(IP는 관리자가 부여하기 때문) (자신의) MAC 주소를 통해 서버에서 (자신의) IP 주소를 알아내는 서비스입니다.

GARP (Gratutious ARP)

:LAN 전체에 자신의 IP와 MAC을 Broadcast로 보내어 ARP Cache를 갱신하도록 알려주는 서비스입니다.
